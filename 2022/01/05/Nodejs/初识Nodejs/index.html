简单介绍Nodejs 内置 file system模块、path 模块以及 http 模块

初识Nodejs

Nodejs 实际上是一种运行环境，我们所学习的JavaScript语言书写出来的代码，一般在浏览器端运行，另一种方式就是在基于Chrome V8引擎的nodejs环境下运行。把JavaScript代码放在浏览器[ 例如：Chrome ]中执行，就是常说的前端开发，而通过nodejs环境运行，就是后端开发，因此JavaScript也可以做后端开发，但这并不常见，因为不如Java等语言具有优势。

fs [ file system ] 模块

nodejs API 中 fs 模块，主要作用就是对文件的读写，有两个函数。首先是引入nodejs API

（1）读文件：readFile(filename[, option], callback)

filename：必选参数，文件名 option：可选参数，表示用哪一种编码格式 callback：必选，回调函数，有两个参数分别用于接收错误信息或读取到的文件内容

（2）写文件：writeFile(filename, context[, option], callback)

filename：必选参数，文件名 context：必选参数，写入文件的内容 option：可选参数，表示用哪一种编码格式 callback：必选，回调函数，有一个参数当写入错误时接收错误信息

path 路径模块

path模块是nodejs用于处理路径的一个API，常用两个方法：

（1）path.join() 用于拼接路径，尽量避免使用 + 直接拼接路径

（2）path.basename(path[, externName]) 可以获取路径的最后一部分，第二个参数可选，写入路径中最后的文件扩展名，则返回结果不带有扩展名

（3）path.extname() 获取路径中文件的扩展名

http 模块

http模块是nodejs官方提供的用来创建web服务器的模块，通过http模块的http.createServer()方法，可以将用户的电脑变成一台服务器，对外提供web资源服务。

前置知识

IP地址是唯一标识每台电脑的一长串点分十进制数字，但是它不便于记忆，因此人们采用了我们常说的Domain域名来表示。每个IP地址对应一个Domain域名地址，所有的IP地址和域名对应关系都存放在一个Domain Name Server【域名服务器】中，用户只需要输入域名即可访问相应网站，如百度服务器IP地址39.156.66.18，而我们不需要记住这种形式IP地址，只需要输入www.baidu.com 即可，域名服务器会帮我们进行IP地址和域名的转换工作。

127.0.0.1 对应的是 localhost，即我们自身电脑的地址。

在之前的学习中，或许出现过我们在自己的电脑上运行不止一个服务器的情况，此时每个web服务器对应不同的端口号，不同的服务器不能使用相同的端口号。

Nodejs Module

nodejs 模块分类

内置模块

内置模块是nodejs官方提供的，例如 fs，http，path

自定义模块

用户在自己电脑上创建的JavaScript文件均属于自定义模块

第三方模块

由第三方开发的便于用户使用的模块，需要下载导入以使用

使用 require

  1. 使用 require 加载自定义模块时，会执行被加载模块的代码。

  2. 模块作用域的好处：防止全局变量污染问题，即如果文件A引入了一个文件B，文件B内有一个变量和文件A中的变量同名，如果不存在模块作用域，则会产生变量冲突[ 使用const或let ]或覆盖[ 使用var ]。

  3. 每个JavaScript文件都内置一个module对象，存储着和当前文件有关的信息，当我们使用require引入一个模块时，实际上接收到的是被引入模块的module.exports对象

  4. nodejs为了简化，设置了exports，它实际指向的对象和module.exports相同，最终还是以module.exports指向的对象为准。由于此，不建议在一个文件中同时使用 exports 和 module.exports

    举例说明 ——> 最终以 module.exports 指向的对象为准

  5. nodejs 遵循 CommonJS 规范，（1）每个模块内部，module变量代表当前模块 （2）module 变量是一个对象，它的exports属性是对外接口 （3）加载某个模块实际上是加载模块的 module.exports 属性

Nodejs NPM

npm包

nodejs中的npm包是由第三方个人或团队开发出来的开源模块，供所有人免费使用，提供了各种便捷的辅助开发功能。在基于nodejs中提供的底层内置模块进行开发时，效率较低。包是经内置模块封装出来的，提供了更高级的API，极大的提高了开发效率。

有一个网站可以搜索包：https://www.npmjs.com/

Node Package Manager 简称 npm包管理工具，当我们下载好nodejs后，就已经安装好了npm包管理工具

npm安装包

如何使用 npm 安装包

npm install xxx 可以简写为 npm i xxx，可以通过 npm i xxx@版本号 来安装指定版本的包，默认下载最新版本包

node_moudles 以及 package-lock.json

当我们通过npm安装包后，会发现项目文件目录下多了 node_modules 文件夹和 package-lock.json 文件，前者存放所有安装到此目录下的包，后者记录 node_modules 目录下每一个包的下载信息，如包的名字，版本号，下载地址等…

包的版本说明

包的版本号用点分十进制表示，如2.24.0，第一位数字改变，表示从底层重构了这个包的代码，进行了大更新；第二位数字改变，表示包中扩展了某些功能；第三位数字改变，表示修复了某些bug。只要前面的数字更新了，后面的数字归零。

包的配置信息 package.json

在项目的根目录必须有一个 package.json 的包管理配置文件，记录与项目有关的信息，如项目的名称、版本号、描述等，项目中用到了哪些包，哪些包在开发阶段使用，哪些包在开发和部署阶段均使用……

在多人协作时，node_modules占用空间极大，影响传输速度，因此一般共享文件时会删除node_modules，由此使得 package.json 作用显现出来，可以根据 package.json 再把需要的 node_modules 中的包安装回来。在通过 npm install xxx 安装包后，package.json 中 dependencies 会记录安装了哪些包，然后我们可以执行一个命令——> npm install 命令，它会一次性将 package.json 中记录的所有依赖包全部下载。

项目开发时依赖包和开发部署时均依赖包的处理

针对于仅在开发时需要依赖的包，我们放在 package.json 的 dependencies 中即可，而对于开发完项目部署上线时仍需要依赖的包，我们将其放到 devDependencies 中。前者使用默认 npm 下载命令即可，后者使用 npm install xxx –save-dev，简写形式为 npm i xxx -D

淘宝镜像下载包

我们有时直接使用npm i xxx会发现下载速度很慢，所以可以使用淘宝镜像。前者的下载是用户直接通过国外服务器下载包必然会有速度降低的情况，而淘宝自己在国内搭建了一个服务器，定时同步国外服务器的第三方包，所以我们也可以直接在国内淘宝服务器下载包。

查看当前npm下载包的源服务器：npm config get registry

切换为淘宝服务器：npm config set registry = https://registry.npm.taobao.org/

这里还有一个切换包镜像源的辅助工具nrm，不过这里暂不介绍了，有兴趣也可去了解了解.
