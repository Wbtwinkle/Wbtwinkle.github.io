<1> 首先是this指向问题，我们每次都在constructor中使用bind手动改变this指向，但如果自定义方法太多，
则显得极其臃肿且不方便，所以我们可以考虑使用箭头函数，并且ES6 class类内部可以直接赋值，作为其

props传递参数

<2> 在<1>中，批量传递参数，使用了…展开运算符，但实际上由于对象不可迭代，因此理论上不能使用…，
如果在正常JavaScript语句中，可以测试得到确实如此。但我们可以使用{…obj}在对对线使用展开运算符时，
外面加一层大括号，可以实现浅复制此对象的功能。而我们在
ReactDOM.render(<MyComponent {…student}/>, document.querySelector(‘#box’))
中，是这样使用的吗？答案是否定的，因为在React中组件标签中，使用单个大括号是为了获取变量值，此处大括号和
JavaScript中的大括号不是一个性质，那么为什么此处没有报错，且成功运行渲染了？因为React结合Babel使得这样
做成为合法的，且只能在组件标签中这样使用。
